# ICS2025 答辩演示操作指南 — 成员 C

## 一、演示前准备（提前 10 分钟）

### 1. 编译系统

```bash
cd /home/tian/ics2025_AB/ics2025/ics2025

# 如果系统自带 as 有问题，先设置：
export PATH=/usr/bin:$PATH

# 一键编译：
bash run.sh --build
```

等待约 1-2 分钟，编译完成后 NEMU 窗口自动弹出。

### 2. 快速验证

在 NEMU 窗口中快速输入以下命令，确认系统正常：
```
login User1
ls /
ps
free
demo
```
确认：demo 输出正常、按 ESC 能停止、shell 恢复正常。

### 3. 环境准备

- 关闭通知（微信/Slack/邮件）
- 插电源，确认电量
- 如果投屏，提前测试分辨率
- NEMU 窗口调到合适大小（不要太满屏，留空间给 PPT 窗口）

## 二、演示流程（12 步，约 5 分钟）

### 第 1 步：启动（10s）

```bash
cd /home/tian/ics2025_AB/ics2025/ics2025
source init.sh
cd nanos-lite && make ARCH=riscv32-nemu run
```

NEMU 窗口出现蓝色框线 Banner。点击窗口确保焦点在 NEMU 上。

> 口播："系统启动后进入多用户登录界面。"

### 第 2 步：登录（5s）

```
login User1
```

### 第 3 步：展示命令（15s）

```
help
ls /
tree /
```

### 第 4 步：系统状态（10s）

```
ps
free
```

### ⭐ 第 5 步：进程调度演示（20s）

```
demo
```

NEMU 窗口中看到三个线程交替输出。让它跑约 5-10 秒，然后：

**按 ESC 键**

确认显示 "=== Demo finished, back to shell ===" 并且 shell 恢复正常。

> 口播："输入 demo，三个线程的日志交替出现证明了 Round-Robin 调度器在正确轮转。按 ESC 停止，shell 正常恢复。"

### ⭐ 第 6 步：C 编译器（15s）

```
cc /test/09-compiler.c
```

> 口播："C 编译器编译并执行包含递归、循环、条件判断的 C 程序，全部输出正确。"

### ⭐ 第 7 步：AI 对话（20s）

```
chat
```

输入：
```
i am tired
sorry
bye
```

> 口播："ELIZA AI 对话，关键词匹配加句式变换，模拟心理治疗师。"

### 第 8 步：文件操作（10s）

```
touch /demo.txt
write /demo.txt Hello!
cat /demo.txt
```

### 第 9 步：收尾（10s）

```
uptime
logout
```

## 三、快捷键参考

| 操作 | 按键 |
|------|------|
| 执行命令 | Enter |
| 删除字符 | Backspace |
| 停止 demo | **ESC**（在 NEMU 窗口中按） |
| 退出 chat | 输入 `bye` 或 `quit` |
| 退出 NEMU | 输入 `shutdown` 或直接关窗口 |

> **注意：** NEMU 没有 Ctrl+C/V！所有命令需手动输入。提前练习几遍。

## 四、故障应急

| 问题 | 解决 |
|------|------|
| NEMU 窗口没弹出 | 检查 SDL 库：`apt list --installed \| grep sdl` |
| 输入字母变大写 | 按一下 Shift 恢复正常 |
| 系统卡住 | 关 NEMU 窗口，重新 `make run` |
| demo 没反应 | 重新编译：`bash run.sh --build` |
| 编译报错 `as --64` | `export PATH=/usr/bin:$PATH` |
| 登录丢失 | 输入 `login User1` 重新登录 |
