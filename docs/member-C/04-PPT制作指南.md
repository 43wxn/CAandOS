# ICS2025 答辩 PPT 制作指南

> 本文档教你如何将 PPT 文稿变成真实的幻灯片。
> 推荐工具：PowerPoint / WPS / Google Slides / Keynote 均可。

---

## 一、整体风格设定

### 配色方案（与终端 8 色调色板一致）

| 用途 | 颜色 | 十六进制 |
|------|------|----------|
| 背景色 | 深海军蓝 | `#0A0E27` |
| 主标题 | 亮青 | `#00D4FF` |
| 正文 | 白 | `#F0F0F0` |
| 强调（成功/运行） | 绿 | `#4AA02C` |
| 警告/就绪 | 黄 | `#E8A317` |
| 错误 | 红 | `#E41B17` |
| 代码块背景 | 深蓝黑 | `#0D1117` |
| 分隔线/次要文字 | 灰 | `#8892B0` |

### 字体

| 用途 | 字体 | 大小 |
|------|------|------|
| 主标题 | 微软雅黑 / Arial Bold | 36-44pt |
| 副标题/章节 | 微软雅黑 / Arial | 24-28pt |
| 正文 | 微软雅黑 / Arial | 18-22pt |
| **代码** | **Consolas / Courier New** | **14-16pt** |
| 注释/脚注 | 微软雅黑 | 12-14pt |

### 布局原则
- 每页一个主题，大量留白
- 代码用深色背景块 + 语法高亮色
- 流程图用矩形框 + 箭头，颜色统一
- 避免动画（答辩时浪费时间）

## 二、逐页制作指南

### 第 1 页 · 封面

- 背景：深海军蓝纯色 `#0A0E27`
- 主标题居中，大号 44pt，白色
- 副标题 24pt，青色 `#00D4FF`
- 底部姓名/日期，灰色 14pt
- 可选：加一条青色细线作为装饰

### 第 2 页 · 团队分工

- 用三个纵向排列的矩形框，每个代表一个成员
- A 和 B 的框用深灰/蓝色（低调），C 的框用青色边框（突出）
- 框内文字：成员名 + 职责
- 框之间用向下箭头 `↓` 连接

### 第 3 页 · 工作全景

- 左右两栏布局
- 左栏标题"课程基线要求"，用白色；列表项用绿色对勾 `✅`
- 右栏标题"⭐ 创新"，用青色 `#00D4FF`；列表项用黄色星号
- 每个列表项不超过一行

### 第 4 页 · ELF 加载器

- 垂直流程图，7 个步骤
- 每个步骤一个圆角矩形，从上到下排列
- 步骤用数字标注 1-5
- 关键数据（0x8FFF0000）用青色高亮

### 第 5 页 · Shell 命令

- 简单的双列表格
- 左列"课程基线"用白色背景，右列"⭐ 扩展"用深蓝背景
- 命令名用等宽字体 Consolas
- 底部一句说明用灰色小字

### 第 6 页 · 进程调度器

- 上半部分：一个终端窗口的模拟图（深色背景 + 绿色文字）
- 模拟图中展示 demo 命令的输出效果
- 下半部分：两行关键代码 `proc_schedule()` 的核心逻辑

### 第 7 页 · 上下文切换

- 整页用代码块风格
- 左侧是 trap.S 汇编代码，右侧是注释说明
- 关键三行（① ② ③）用青色高亮
- 底部画一个简单的 Context 结构体布局图

### 第 8 页 · Mini C 编译器

- 水平四阶段流水线图
- 四个矩形框从左到右排列：词法分析 → 语法解析 → 代码生成 → VM
- 框之间用粗箭头连接
- 每个框内列出关键数据（Token 数/指令数等）

### 第 9 页 · 编译器能力证明

- 左侧放 C 源码（等宽字体，深色背景，语法高亮）
- 右侧放运行结果（绿色文字 `7 5 13 1 0 55`，大号）
- 底部标注：`cc /test/09-compiler.c` 命令

### 第 10 页 · ELIZA AI

- 左侧放一个聊天界面模拟图（深色背景）
- 用户消息和 AI 回复用不同颜色（用户青色，AI 绿色）
- 右侧放关键词匹配规则的简单示意图

### 第 11 页 · 多用户与状态栏

- 上半部分：多用户系统架构图（三列用户 + 底部共享层）
- 下半部分：状态栏截图或模拟图
- 状态栏的颜色严格使用实际配色

### 第 12 页 · 界面美化

- 3×3 的九宫格布局
- 每个格子里放改造前后对比（Before→After）
- 用绿色箭头表示改进方向

### 第 13 页 · Bug 修复

- 6×2 的表格
- Bug 描述简洁，根因用红色标注
- 可选：每个 Bug 加一个小图标

### 第 14 页 · 测试程序

- 两列 5 行的表格
- 每行：编号 + 测试内容
- 底部标注运行方式

### 第 15 页 · 技术指标

- 数据用大号数字展示（视觉冲击力）
- 30+ / 22 / 3 / 10 / 12 等数字用青色大字体
- 布局：3 行 × 3 列

### 第 16 页 · 演示菜单

- 一个终端模拟图
- 命令序列从上到下排列
- 关键命令（demo/cc/chat）用星号标注

### 第 17 页 · 总结

- 左右两栏
- 左栏：技术成长（绿色箭头列表）
- 右栏：诚实说明（黄色背景框）

### 第 18 页 · 致谢

- 简洁大标题"谢谢！"
- 底部小字：欢迎提问 + 可演示内容列表

## 三、代码展示技巧

### 代码块格式（适用于 PPT 第 4/7/8 页）

在 PPT 中插入代码的方法：
1. 插入 → 形状 → 矩形
2. 填充色：`#0D1117`（GitHub 风格深色背景）
3. 插入文本框，字体 Consolas 14pt
4. 关键字用颜色区分：青色 `int/if/while/return`，黄色 `数字`，绿色 `字符串`

### 流程图绘制（适用于 PPT 第 4/8 页）

1. 插入 → 形状 → 矩形/圆角矩形
2. 填充色：`#16213E`，边框色：`#00D4FF`
3. 连接线用箭头形状，颜色 `#8892B0`
4. 文字白色，居中

## 五、PPT 制作 Checklist

- [ ] 统一背景色 `#0A0E27` 应用到所有页
- [ ] 标题统一 36pt 白色
- [ ] 代码全部用 Consolas 14pt
- [ ] 颜色方案一致（青/绿/黄/红 四个强调色）
- [ ] 每页文字不超过 7 行（除代码页）
- [ ] 没有动画（减少答辩等待时间）
- [ ] 页码标注
- [ ] 图片不拉伸变形
- [ ] 文件大小 < 20MB（避免拷贝问题）
- [ ] 备份到 U 盘 + 云端
