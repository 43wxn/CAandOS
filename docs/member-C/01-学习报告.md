## 四、关键代码片段

### 4.1 系统调用全链路

```
用户: _syscall_(SYS_write, fd, buf, len)
  → do_syscall(): a7=4, a0=fd, a1=buf, a2=len
  → ecall
  → trap.S: 保存 32GPR + mcause/mstatus/mepc (144 bytes)
  → cte.c: c->mepc+=4, 识别 EVENT_SYSCALL
  → irq.c: do_syscall(c)
  → syscall.c: switch(a7) → c->GPRx = fs_write(fd,buf,len)
  → trap.S: csrw mepc → mret
  → 用户: a0 = 写入字节数
```

### 4.2 Round-Robin 调度器

```c
Context *proc_schedule(Context *prev) {
  current->cp = prev;                          // 保存当前 Context
  current->state = PROC_RUNNABLE;              // 标记就绪
  static int last_idx = 0;
  for (int n = 0; n < MAX_NR_PROC; n++) {      // 遍历 PCB 数组
    int i = (last_idx + n) % MAX_NR_PROC;
    if (pcb[i].state == PROC_RUNNABLE) {
      last_idx = (i + 1) % MAX_NR_PROC;
      if (&pcb[i] != current) {                // 跳过当前进程
        pcb[i].state = PROC_RUNNING;
        current = &pcb[i];
        return pcb[i].cp;                       // 返回目标 Context*
      }
    }
  }
  current->state = PROC_RUNNING;               // 无其他进程，继续当前
  return prev;
}
```

### 4.3 fork() 实现

```c
int proc_fork(Context *c) {
  int slot = 找空闲 PCB;
  void *stack = new_page(4);  // 16KB 独立栈
  Context *child = (Context *)stack;
  for (int i = 0; i < 32; i++) child->gpr[i] = c->gpr[i];  // 逐字段拷贝
  child->mepc = c->mepc;
  child->gpr[2] = stack.end;   // 独立栈指针
  child->GPRx = 0;             // 子进程 fork() 返回 0
  pcb[slot] = { .cp=child, .state=PROC_RUNNABLE, ... };
  return pcb[slot].pid;
}
```

### 4.4 调度器演示（demo 命令）

```
demo 命令执行流程:
  proc_start_demo()
    → 创建 logger/worker/watchdog (pid 10/11/12)
    → while(demo_running):
        yield()           ← 让线程运行

```

**Q4: ELIZA AI 原理？**
> MIT 1966 年 Weizenbaum 经典算法——关键词模式匹配 + 句式变换。10+ 种情感模式，7 种回退回复，50 轮对话上限。

**Q5: C 编译器如何处理 if/while？**
> 可修补跳转——if 生成 OP_JMP_FALSE 占位，解析完条件块后回填真实地址。while 先记录循环起始位置，条件假时跳过，循环体结束无条件跳回。
